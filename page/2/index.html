<div class="row">
    <p>在fedora 22上面试用了一下docker，发现与ubuntu 有以下区别，/var/run/docker.sock这个文件是属于用户root,组root的，这样就需要每次在运行docker 的时候必须给root权限，所以我修改了/var/run/docker.sock的组</p>
    <p>docker.sock 是在守护进程启动的时候穿件的，所以要在fedora每次启动时都要修改</p>
     highlight bash
    sudo touch /etc/rc.d/rc.local
    sudo vim /etc/rc.d/rc.local 添加 sudo systemctl start docker 
    sudo chmod a+x /etc/rc.d/rc.local

    sudo vim /usr/lib/systemd/system/rc-local.service
    添加
        [Install]
        WantedBy=multi-user.target
    sudo systemctl enable rc-local.service

    sudo chgrp dockerroot /var/run/docker.sock
    <p>这里用dockerroot (ubuntu 是docker)是用dnf(yum)安装后多出的用户组</p>
    <p>fedora 和ubuntu 的一些区别，比如 全好xx.h，那么需要安装的名字为 libxx.dev, libxx.devel, python-xx, xx-dev, xx-devel, python-xx-</p>

2.1 打开docker bash
docker run xx:latest -t -i /bin/bash

apt-get install openssh-server #安装ssh
#需要修改/etc/sshd/sshd_config文件中内容
PermitRootLogin yes  
UsePAM no

2.2 启动ssh，容器以后台方式运行

docker run -d -p 50001:22 <容器id> /usr/sbin/sshd-D  
#容器id可通过 docker ps-a查看，最上面的为最新的。

2.3 通过ssh连接到容器安装软件

ssh root@127.0.0.1-p 50001  
#连上后想装什么就装什么，可使用exit退出容器，但后台还会运行。

2.4 服务安装完成后，停止容器。

docker stop <容器id> #停止运行的容器

2.5 把容器提交生成最新的镜像

docker commit <容器id> debian02 #把这个容器提交生成新的debian02镜像(
